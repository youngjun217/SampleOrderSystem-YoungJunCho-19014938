# PRD — 반도체 시료 생산주문관리 시스템
**S Semi 사내 프로젝트 | 작성일: 2026-05-08**

---

## 1. 배경 및 문제 정의

S Semi는 다양한 반도체 시료를 생산하여 연구소, 팹리스 업체, 대학 연구실 등에 납품하는 회사다.
시료는 고객 주문이 접수되면 웨이퍼 공정 설비를 통해 제작되고, 검수를 거쳐 출고된다.

최근 주문량이 급증하면서 **엑셀·메모장 기반 수작업 관리**의 한계가 드러났다.

| 증상 | 원인 |
|------|------|
| "이 주문 처리됐나요?" 반복 문의 | 주문 상태 추적 불가 |
| 완성 예정 시점 불명확 | 공정 일정 가시성 부재 |
| 재고 있는데 공정이 돌아감 | 재고-주문 연계 미흡 |

---

## 2. 목표

- 주문부터 출고까지 전 과정을 **단일 시스템**에서 관리한다.
- 재고를 실시간으로 확인하여 **불필요한 공정 착수를 방지**한다.
- 담당자가 언제든 **주문·공정·재고 현황을 한눈에** 파악할 수 있게 한다.

---

## 3. 사용자 (Stakeholders)

| 역할 | 설명 |
|------|------|
| 주문 담당자 | 고객 주문 접수·조회·상태 관리 |
| 공정 엔지니어 | 공정 일정 등록, 진행 상태 업데이트 |
| 재고 담당자 | 시료 재고 입출고 및 현황 관리 |
| 관리자 (Admin) | 전체 현황 모니터링, 더미 데이터 생성 |

---

## 4. 기능 요구사항

### 4.1 주문 관리
- 고객명, 시료 종류, 수량, 납기일을 입력하여 주문을 생성한다.
- 주문 상태는 `접수 → 재고확인 → 공정대기 → 공정중 → 검수 → 출고완료` 단계로 관리한다.
- 주문 목록 조회, 단건 상세 조회, 상태 변경, 취소가 가능하다.

### 4.2 재고 관리
- 시료 종류별 현재 재고 수량을 관리한다.
- 주문 접수 시 재고를 먼저 확인하여, 재고가 충분하면 공정 없이 즉시 출고 처리한다.
- 재고 입고(공정 완료) 및 출고(주문 처리) 이력을 기록한다.

### 4.3 공정 관리
- 재고가 부족한 경우 공정을 예약한다.
- 공정 항목은 시료 종류, 목표 수량, 착수일, 예상 완료일, 진행 상태를 포함한다.
- 공정 상태: `대기 → 진행중 → 완료`
- 공정 완료 시 자동으로 재고에 반영된다.

### 4.4 현황 모니터링 (DataMonitor 연동)
- 전체 주문 수, 상태별 건수, 총 매출을 콘솔 대시보드로 표시한다.
- 진행 중인 공정 목록과 예상 완료일을 실시간으로 표시한다.
- 재고 부족 시료 목록을 강조 표시한다.

### 4.5 더미 데이터 생성 (DummyDataGenerator 연동)
- 테스트용 주문·재고·공정 데이터를 일괄 생성한다.
