# PRD — 반도체 시료 생산주문관리 시스템
**S Semi 사내 프로젝트 | 작성일: 2026-05-08**

---

## 1. 배경 및 문제 정의

S Semi는 다양한 반도체 시료를 생산하여 연구소, 팹리스 업체, 대학 연구실 등에 납품하는 회사다.
시료는 고객 주문이 접수되면 웨이퍼 공정 설비를 통해 제작되고, 검수를 거쳐 출고된다.

최근 주문량이 급증하면서 **엑셀·메모장 기반 수작업 관리**의 한계가 드러났다.

| 증상 | 원인 |
|------|------|
| "이 주문 처리됐나요?" 반복 문의 | 주문 상태 추적 불가 |
| 완성 예정 시점 불명확 | 생산라인 배정 및 일정 가시성 부재 |
| 재고 있는데 생산라인이 돌아감 | 재고-주문 연계 미흡 |

---

## 2. 시스템 개요

- **콘솔 기반 CLI**: 담당자가 직접 명령을 입력하여 시스템을 조작한다.
- **핵심 기능**: 시료 등록, 주문 접수 및 처리, 생산라인 배정 및 현황 관리.
- **수요 기반 생산**: 생산라인은 주문이 들어온 시료에 대해서만 가동한다. 사전 생산(예비 생산)은 하지 않는다.
- **생산라인 단위**: 하나의 생산라인은 시료를 하나씩 순차 생산한다. 동시에 하나의 주문만 처리할 수 있다.

---

## 3. 목표

- 주문부터 출고까지 전 과정을 **단일 시스템**에서 관리한다.
- 재고를 확인하여 **불필요한 생산라인 가동을 방지**한다.
- 담당자가 언제든 **주문·생산라인·재고 현황을 한눈에** 파악할 수 있게 한다.

---

## 4. 사용자 (Stakeholders)

| 역할 | 시스템 접근 | 주요 책임 |
|------|------------|----------|
| 고객 | 외부 (이메일) | 필요한 시료 종류·수량을 이메일로 요청 |
| 주문담당자 | 콘솔 직접 사용 | 이메일 요청을 바탕으로 주문서 작성 및 현황 조회 |
| 생산담당자 | 콘솔 직접 사용 | 시료 등록, 주문 승인/거절, 생산라인 배정·관리 |

> 고객은 시스템에 직접 접근하지 않는다. 이메일 요청을 주문담당자가 대신 시스템에 입력한다.

## 5. 역할별 흐름

### 고객
```
필요 시료 확인 → 이메일 작성 (시료 종류, 수량, 납기 요청) → 주문담당자에게 발송
```

### 주문담당자
```
이메일 수신 → [콘솔] 주문서 작성 (고객명, 시료 종류, 수량, 납기일 입력)
           → 주문 목록 조회 및 현황 관리
           → 생산담당자의 승인/거절 결과 확인
```

### 생산담당자
```
[시료 등록]
  새로운 시료 개발 완료 → [콘솔] 시료 정보 등록 (시료명, 규격)

[주문 처리]
  주문 수신 → 재고 확인
    ├─ 재고 충분 → 주문 승인 → 즉시 출고
    ├─ 재고 부족 → 주문 승인 → 유휴 생산라인 배정
    │               → 생산라인이 시료를 하나씩 생산
    │               → 생산 완료 → 재고 반영 → 출고
    └─ 요청 불가 (시료 미등록, 납기 불가 등) → 주문 거절 (사유 기재)

[생산라인 관리]
  생산라인 상태 조회 (유휴/가동 중)
  → 생산 완료 처리 (완료 입력 시 재고 자동 반영)
```

### 전체 흐름 요약
```
고객 ──(이메일)──▶ 주문담당자 ──(주문 등록)──▶ 생산담당자
                                                    │
                                승인 ◀──────────────┤
                                거절 ◀──────────────┘
                                  │
                      재고 있음 ──┤──────────────────────▶ 출고
                                  │
                      재고 없음 ──┘──▶ 생산라인 배정
                                          │
                                    시료 하나씩 순차 생산
                                          │
                                    생산 완료 → 재고 반영 ──▶ 출고
```

## 6. 기능 요구사항

### 6.1 시료 관리 (생산담당자)
- 새로운 시료를 시스템에 등록한다 (시료명, 규격).
- 시료 목록 조회 및 현재 재고 수량 확인이 가능하다.

### 6.2 주문 관리 (주문담당자)
- 고객명, 시료 종류, 수량, 납기일을 입력하여 주문을 생성한다.
- 주문 상태는 아래 5단계로 관리한다.

  | 상태 | 코드 | 설명 |
  |------|------|------|
  | 주문접수 | `RESERVED` | 주문담당자가 주문을 등록한 초기 상태 |
  | 주문거절 | `REJECTED` | 생산담당자가 주문을 거절한 상태 (사유 포함) |
  | 생산중 | `PRODUCING` | 주문 승인 완료, 재고 부족으로 생산라인 가동 중 |
  | 출고대기 | `CONFIRMED` | 주문 승인 완료, 재고 확보되어 출고 대기 중 |
  | 출고완료 | `RELEASE` | 시료가 고객에게 출고된 최종 상태 |

  ```
  RESERVED
    ├─ 거절 ──────────────────────────────▶ REJECTED
    ├─ 승인 + 재고 부족 ──▶ PRODUCING ──▶ CONFIRMED ──▶ RELEASE
    └─ 승인 + 재고 충분 ──────────────▶ CONFIRMED ──▶ RELEASE
  ```

- 주문 목록 조회, 단건 상세 조회가 가능하다.

### 6.3 주문 승인/거절 (생산담당자)
- 접수된 주문 목록에서 승인 또는 거절 처리한다.
- 거절 시 사유를 입력한다.
- 승인 시 재고를 자동 확인하여 즉시 출고 또는 생산라인 배정으로 분기한다.

### 6.4 생산라인 관리 (생산담당자)
- 생산라인 목록과 현재 상태(유휴/가동 중)를 조회한다.
- 생산라인은 한 번에 하나의 주문만 처리하며, 시료를 하나씩 순차 생산한다.
- 유휴 생산라인이 없으면 주문은 대기 상태로 전환된다.
- 생산 완료 처리 시 생산 수량이 재고에 자동 반영되고, 연결된 주문은 출고 처리된다.

### 6.5 현황 모니터링
- 전체 주문 수, 상태별 건수를 콘솔 대시보드로 표시한다.
- 생산라인별 가동 상태와 담당 주문을 표시한다.
- 재고 현황(시료별 수량)을 표시한다.

### 6.6 더미 데이터 생성 (테스트용)
- 테스트용 시료·주문·생산라인 데이터를 일괄 생성한다.
