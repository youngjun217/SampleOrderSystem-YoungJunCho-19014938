# PRD — 반도체 시료 생산주문관리 시스템
**S Semi 사내 프로젝트 | 작성일: 2026-05-08 | 최종수정: 2026-05-08**

---

## 1. 배경 및 문제 정의

S Semi는 다양한 반도체 시료를 생산하여 연구소, 팹리스 업체, 대학 연구실 등에 납품하는 회사다.
시료는 고객 주문이 접수되면 웨이퍼 공정 설비를 통해 제작되고, 검수를 거쳐 출고된다.

최근 주문량이 급증하면서 **엑셀·메모장 기반 수작업 관리**의 한계가 드러났다.

| 증상 | 원인 |
|------|------|
| "이 주문 처리됐나요?" 반복 문의 | 주문 상태 추적 불가 |
| 완성 예정 시점 불명확 | 생산라인 배정 및 일정 가시성 부재 |
| 재고 있는데 생산라인이 돌아감 | 재고-주문 연계 미흡 |

---

## 2. 시스템 개요

- **CLI (콘솔)**: 담당자가 직접 명령을 입력하여 시스템을 조작한다. ANSI 컬러 + 박스 드로잉 UI.
- **GUI (데스크톱)**: PyQt6 기반 다크 테마 애플리케이션. 사이드바 탐색, 테이블, 다이얼로그 제공.
- **수요 기반 생산**: 생산라인은 주문이 들어온 시료에 대해서만 가동한다.
- **생산라인 단위**: 하나의 생산라인은 시료를 하나씩 순차 생산하며, 동시에 하나의 주문만 처리한다.

---

## 3. 목표

- 주문부터 출고까지 전 과정을 **단일 시스템**에서 관리한다.
- 재고를 확인하여 **불필요한 생산라인 가동을 방지**한다.
- 담당자가 언제든 **주문·생산라인·재고 현황을 한눈에** 파악할 수 있게 한다.

---

## 4. 사용자 (Stakeholders)

| 역할 | 시스템 접근 | 주요 책임 |
|------|------------|----------|
| 고객 | 외부 (이메일) | 필요한 시료 종류·수량을 이메일로 요청 |
| 주문담당자 | CLI / GUI | 이메일 요청을 바탕으로 주문서 작성 및 현황 조회 |
| 생산담당자 | CLI / GUI | 시료 등록, 주문 승인/거절, 생산라인 배정·관리 |

> 고객은 시스템에 직접 접근하지 않는다. 이메일 요청을 주문담당자가 대신 시스템에 입력한다.

## 5. 역할별 흐름

### 고객
```
필요 시료 확인 → 이메일 작성 (시료 종류, 수량, 납기 요청) → 주문담당자에게 발송
```

### 주문담당자
```
이메일 수신 → [시스템] 주문서 작성 (고객명, 시료 ID, 수량 입력)
           → 주문 목록 조회 및 현황 관리
           → 생산담당자의 승인/거절 결과 확인
           → CONFIRMED 주문 출고 실행
```

### 생산담당자
```
[시료 등록]
  새로운 시료 → [시스템] 시료 등록 (시료ID, 이름, 평균생산시간, 수율)

[주문 처리]
  주문 수신 → 재고 확인
    ├─ 재고 충분 → 주문 승인 → CONFIRMED (즉시 출고 대기)
    ├─ 재고 부족 → 주문 승인 → 유휴 생산라인 배정 → PRODUCING
    │               → 생산 완료 → 재고 반영 → CONFIRMED → 출고
    └─ 요청 불가 → 주문 거절 (사유 기재) → REJECTED

[생산라인 관리]
  생산라인 상태 조회 (유휴/가동 중)
  → 생산 완료 처리 → 재고 자동 반영 → 다음 대기 작업 자동 배정 (FIFO)
```

### 전체 흐름 요약
```
고객 ──(이메일)──▶ 주문담당자 ──(주문 등록)──▶ 생산담당자
                                                    │
                                승인 ◀──────────────┤
                                거절 ◀──────────────┘
                                  │
                      재고 있음 ──┤──▶ CONFIRMED ──▶ [출고처리] ──▶ RELEASE
                                  │
                      재고 없음 ──┘──▶ 생산라인 배정 (FIFO)
                                          │
                                    시료 순차 생산
                                          │
                                    완료 → 재고 반영 ──▶ CONFIRMED ──▶ RELEASE
```

## 6. 기능 요구사항

### 6.1 시료 관리
- 새로운 시료를 시스템에 등록한다.
  - 입력값: **시료 ID** (사용자 정의, 예: GaN-001), **이름**, **평균 생산시간(h)**, **수율(0.0~1.0)**
- 시료 목록 조회 (ID/이름 검색 포함) 및 현재 재고 수량 확인.

### 6.2 주문 관리
- 고객명, 시료 ID, 수량을 입력하여 주문을 생성한다 (상태: `RESERVED`).
- 주문 상태는 아래 5단계로 관리한다.

  | 상태 | 코드 | 설명 |
  |------|------|------|
  | 주문접수 | `RESERVED` | 주문담당자가 등록한 초기 상태 |
  | 주문거절 | `REJECTED` | 생산담당자가 거절한 상태 (사유 포함) |
  | 생산중 | `PRODUCING` | 승인 완료, 재고 부족으로 생산라인 가동 중 |
  | 출고대기 | `CONFIRMED` | 승인 완료, 재고 확보되어 출고 대기 중 |
  | 출고완료 | `RELEASE` | 시료가 고객에게 출고된 최종 상태 |

  ```
  RESERVED
    ├─ 거절 ──────────────────────────────▶ REJECTED
    ├─ 승인 + 재고 부족 ──▶ PRODUCING ──▶ CONFIRMED ──▶ RELEASE
    └─ 승인 + 재고 충분 ──────────────▶ CONFIRMED ──▶ RELEASE
  ```

- 주문 목록 조회, 상태별 필터 조회가 가능하다.

### 6.3 주문 승인/거절
- 접수된 주문(RESERVED) 목록에서 승인 또는 거절 처리한다.
- 거절 시 사유를 입력한다.
- **승인 시 자동 재고 분기**:
  - 재고 충분 → `CONFIRMED` (재고 즉시 차감)
  - 재고 부족 → `PRODUCING` + 생산라인 자동 배정

### 6.4 생산라인 관리
- 생산라인 목록과 현재 상태(유휴/가동 중)를 조회한다.
- 생산라인은 한 번에 하나의 주문만 처리하며, 시료를 하나씩 순차 생산한다.
- 유휴 생산라인이 없으면 주문은 대기 큐에 등록된다 (**FIFO 스케줄링**).
- 생산 완료 처리 시:
  - 생산량 계산: `실 생산량 = ceil(부족분 / (수율 × 0.9))`
  - `총 생산시간 = 평균생산시간 × 실 생산량`
  - 재고 반영 후 주문 상태 `PRODUCING → CONFIRMED`
  - 유휴 라인에 다음 대기 작업 자동 배정

### 6.5 출고 처리
- `CONFIRMED` 상태 주문 목록을 조회한다 (접수 순 정렬).
- 특정 주문을 선택하여 출고를 실행한다: `CONFIRMED → RELEASE`

### 6.6 모니터링
- 전체 주문 수, 상태별 건수를 대시보드로 표시한다.
- 생산라인별 가동 상태, 담당 주문, 완료 예정 시간을 표시한다.
- 시료별 재고 현황 및 여유/부족/고갈 상태를 표시한다.
  - **여유**: 재고 ≥ RESERVED 대기 수량
  - **부족**: 재고 > 0 but < RESERVED 대기 수량
  - **고갈**: 재고 = 0

### 6.7 더미 데이터 생성 (테스트용)
- 테스트용 시료·주문·생산라인 데이터를 일괄 생성한다.
- `--reset`, `--list`, `--summary` 플래그 지원.

## 9. 데이터 모델

```
Sample       (시료)
  id                  : str   # 사용자 정의 시료 ID (예: GaN-001)
  name                : str
  avg_production_time : float # 평균 생산시간 (h)
  yield_rate          : float # 수율 (0.0 ~ 1.0)
  stock_quantity      : int
  updated_at          : str

Order        (주문)
  id                  : str
  customer_name       : str
  sample_id           : str
  sample_name         : str
  quantity            : int
  due_date            : str
  status              : OrderStatus
  reject_reason       : str
  created_at          : str

ProductionLine (생산라인)
  id                  : str
  name                : str
  status              : LineStatus (IDLE / RUNNING)
  current_order_id    : str

Production   (생산 이력)
  id                  : str
  line_id             : str
  order_id            : str
  sample_id           : str
  sample_name         : str
  quantity            : int   # 주문 수량
  shortage            : int   # 부족분 = 주문수량 - 재고
  actual_quantity     : int   # 실 생산량 = ceil(shortage / (yield × 0.9))
  total_time          : float # 총 생산시간 (h)
  status              : ProductionStatus (WAITING / RUNNING / DONE)
  queued_at           : str   # FIFO 정렬 기준
  started_at          : str
  completed_at        : str
```
