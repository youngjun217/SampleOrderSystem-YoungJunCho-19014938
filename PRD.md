# PRD — 반도체 시료 생산주문관리 시스템
**S Semi 사내 프로젝트 | 작성일: 2026-05-08**

---

## 1. 배경 및 문제 정의

S Semi는 다양한 반도체 시료를 생산하여 연구소, 팹리스 업체, 대학 연구실 등에 납품하는 회사다.
시료는 고객 주문이 접수되면 웨이퍼 공정 설비를 통해 제작되고, 검수를 거쳐 출고된다.

최근 주문량이 급증하면서 **엑셀·메모장 기반 수작업 관리**의 한계가 드러났다.

| 증상 | 원인 |
|------|------|
| "이 주문 처리됐나요?" 반복 문의 | 주문 상태 추적 불가 |
| 완성 예정 시점 불명확 | 공정 일정 가시성 부재 |
| 재고 있는데 공정이 돌아감 | 재고-주문 연계 미흡 |

---

## 2. 목표

- 주문부터 출고까지 전 과정을 **단일 시스템**에서 관리한다.
- 재고를 실시간으로 확인하여 **불필요한 공정 착수를 방지**한다.
- 담당자가 언제든 **주문·공정·재고 현황을 한눈에** 파악할 수 있게 한다.

---

## 3. 사용자 (Stakeholders)

| 역할 | 시스템 접근 | 주요 책임 |
|------|------------|----------|
| 고객 | 외부 (이메일) | 필요한 시료 종류·수량을 이메일로 요청 |
| 주문담당자 | 시스템 직접 사용 | 이메일 요청을 바탕으로 주문서 작성 및 관리 |
| 생산담당자 | 시스템 직접 사용 | 개발시료 등록, 주문 승인/거절, 공정·재고 관리 |

> 고객은 시스템에 직접 접근하지 않는다. 이메일 요청을 주문담당자가 대신 시스템에 입력한다.

---

## 4. 역할별 흐름

### 고객
```
필요 시료 확인 → 이메일 작성 (시료 종류, 수량, 납기 요청) → 주문담당자에게 발송
```

### 주문담당자
```
이메일 수신 → 시스템에 주문서 작성 (고객명, 시료 종류, 수량, 납기일 입력)
           → 주문 목록 조회 및 현황 관리
           → 생산담당자의 승인/거절 결과 확인
```

### 생산담당자
```
[시료 등록]
  개발 완료된 새 시료 → 시스템에 시료 정보 등록 (종류, 초기 재고)

[주문 처리]
  주문 수신 → 재고 확인
    ├─ 재고 충분 → 주문 승인 → 즉시 출고 처리
    ├─ 재고 부족 → 주문 승인 → 공정 예약 → 공정 진행 → 재고 입고 → 출고
    └─ 요청 불가 (시료 미존재, 납기 불가 등) → 주문 거절 (사유 기재)
```

### 전체 흐름 요약
```
고객 ──(이메일)──▶ 주문담당자 ──(주문 등록)──▶ 생산담당자
                                                  │
                              승인 ◀──────────────┤
                              거절 ◀──────────────┘
                                │
                    재고 있음 ──┤──▶ 출고
                    재고 없음 ──┘──▶ 공정 착수 ──▶ 재고 입고 ──▶ 출고
```

## 5. 기능 요구사항

### 5.1 시료 관리 (생산담당자)
- 새로운 시료 종류를 시스템에 등록한다 (시료명, 규격, 초기 재고).
- 시료 목록 조회 및 현재 재고 수량 확인이 가능하다.

### 5.2 주문 관리 (주문담당자)
- 고객명, 시료 종류, 수량, 납기일을 입력하여 주문을 생성한다.
- 주문 상태는 `접수 → 승인대기 → 승인/거절 → 공정대기 → 공정중 → 출고완료` 단계로 관리한다.
- 주문 목록 조회, 단건 상세 조회가 가능하다.

### 5.3 주문 승인/거절 (생산담당자)
- 접수된 주문을 목록에서 확인하고 승인 또는 거절 처리한다.
- 거절 시 사유를 입력한다.
- 승인 시 재고를 자동 확인하여 출고 또는 공정 예약으로 분기한다.

### 5.4 공정 관리 (생산담당자)
- 재고 부족 시 공정이 자동 예약된다.
- 공정 상태를 `대기 → 진행중 → 완료`로 수동 업데이트한다.
- 공정 완료 시 해당 수량이 재고에 반영된다.

### 5.5 현황 모니터링
- 전체 주문 수, 상태별 건수를 콘솔 대시보드로 표시한다.
- 진행 중인 공정 목록과 예상 완료일을 표시한다.
- 재고 부족 시료 목록을 표시한다.

### 5.6 더미 데이터 생성 (관리자/테스트용)
- 테스트용 주문·재고·공정 데이터를 일괄 생성한다.
